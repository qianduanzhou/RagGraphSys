# 学习路线（小白入门）

> 这是一份给 **Python 和 LangChain 新手** 的学习地图。本项目用 LangChain + LangGraph 搭了一个「知识库检索问答」系统。跟着下面 8 步走，你能从零看懂整套代码。
> 配套食用：[langchain-langgraph-用法清单](langchain-langgraph-用法清单.md)（每个 API 在哪用、第几步）。

## 先搞懂：LangChain 和 LangGraph 是什么关系？

新手最容易把这两个名字搞混。它们是**两个不同的库，分工不同**：

- **LangChain = 组件库**。它给你「积木」：怎么调大模型对话（`ChatOpenAI`）、怎么把文本变成向量（`OpenAIEmbeddings`）、怎么把长文档切成小块（`RecursiveCharacterTextSplitter`）、消息怎么表示（`SystemMessage`/`HumanMessage`）。一句话：**它管「单个能力怎么做」**。
- **LangGraph = 编排引擎**。它给你「胶水」：把一堆积木组织成一张**带状态的流程图**，节点之间可以并行、循环、按条件分叉、边跑边流式输出。一句话：**它管「多个能力怎么串起来协作」**。

打个比方：LangChain 是各种零件（发动机、轮子、方向盘），LangGraph 是把它们装成一辆能跑的车。本项目第 1~3 步只碰 LangChain（单个零件），第 4 步开始用 LangGraph 把零件组装成车。

## 第 0 步 · 前置知识（建议先过一遍）

如果你对 Python 基础、命令行、HTTP 完全陌生，下面这点底子够用了。不用全学会，**遇到不懂的回头查就行**。

### 0.1 Python 基础

- **虚拟环境（venv）**：Python 项目习惯把依赖装在一个隔离的「虚拟环境」里，避免污染系统。本项目用 `backend\.venv\`。命令是 `python -m venv .venv` 创建、`.\.venv\Scripts\Activate.ps1` 激活、`pip install -r requirements.txt` 装依赖。
  - 官方教程：https://docs.python.org/zh-cn/3/tutorial/venv.html
- **pip**：Python 的包管理器，`pip install 包名` 装第三方库（比如 `pip install langchain`）。
- **包与 `__init__.py`**：一个文件夹里有 `__init__.py` 就算一个「包」，可以用 `from services.llm_service import LLMService` 这样跨文件导入。本项目的 `backend\` 就是根包。
- **类型提示**：函数参数和返回值后面跟的类型，如 `def embed(text: str) -> List[float]`。它**不强制运行**，只是给人读、给编辑器提示用的。本项目大量用到：
  - `Optional[X]` = 可以是 X 也可以是 None；
  - `List[X]` = 存 X 的列表；
  - `TypedDict` = 一个字典，但规定了它有哪些键、每个键什么类型（LangGraph 的状态就是用它定义的）。
- 官方 Python 入门：https://docs.python.org/zh-cn/3/tutorial/index.html

### 0.2 命令行

本项目启动要跑几条命令，认识它们就行：

- `docker compose up -d`：用 Docker 起依赖服务（Qdrant 向量库、Neo4j 图数据库）。Docker 把这些服务打包好，一行命令拉起。
  - 入门：https://docs.docker.com/get-started/
- `uvicorn main:app --reload`：启动后端 API 服务（FastAPI 框架）。`uvicorn` 是服务器、`main:app` 指向 `main.py` 里的 `app` 变量。
  - FastAPI 文档：https://fastapi.tiangolo.com/zh/tutorial/
- `npm run dev`：启动前端开发服务器（如果用了前端）。

### 0.3 HTTP 与 SSE

- **HTTP 请求**：浏览器/前端用 `POST /api/chat` 这样的请求调用后端，带上 JSON 数据（你的问题），后端返回 JSON（答案）。GET/POST 是两种最常见的请求方法。
  - MDN 入门：https://developer.mozilla.org/zh-CN/docs/Web/HTTP/Overview
- **SSE（Server-Sent Events）**：服务器「单向、持续」地把数据一段段推给浏览器。本项目流式对话用它：后端边生成答案边把字一个个推过来，前端像打字机一样显示，不用等全部生成完。
  - MDN SSE：https://developer.mozilla.org/zh-CN/docs/Web/API/Server-sent_events/Using_server-sent_events

## 三阶段总览

| 阶段 | 步骤 | 主题 | 你会学到 |
|------|------|------|----------|
| **阶段一 · LangChain 基础** | 步 1~3 | 单个组件怎么用 | 对话模型、向量化、文本切分 |
| **阶段二 · LangGraph 单路图** | 步 4~6 | 把组件连成一张图 | 状态与节点、构图编译、同步/流式调用 |
| **阶段三 · 多智能体** | 步 7~8 | 并行多助手协作 | 多智能体节点、fan-out/fan-in 并行汇合 |

建议总时长：有编程基础约 **6~10 小时**，纯新手可放宽到 **2~3 天**，每个文件读完后对照源码敲一遍。

## 数据流总览图

先建立全局印象，再逐步拆。下面这张图是单路 RAG 图（步 4~6）的完整数据流，标注了每一环用到的 API 和对应第几步：

```
用户问题
   |
   v
[router_node]  决定是否检索                     (步4 节点 / 步5 add_edge)
   |
   | add_conditional_edges → 返回 ["qdrant_node","neo4j_node"] (并行扇出)
   v
+------------------+     +------------------+
| qdrant_node      |     | neo4j_node       |
| 向量检索          |     | 图谱检索          |       (步2 embed_query / 步1 extract_keywords)
| (OpenAIEmbeddings)|    | (ChatOpenAI 抽词) |
+--------+---------+     +--------+---------+
         |                        |
         +-----------+------------+
                     |  两条边都指向 merge = 汇合等待 (fan-in)
                     v
              [merge_node]  融合两路结果成 context        (步4 节点 / 步5 add_edge)
                     |
                     v
              [llm_node]  生成回答                         (步1 invoke/stream + 步4 get_stream_writer)
                     |
         add_conditional_edges → streaming? 
         |-- 流式 → END (直接结束，边生成边推字)
         |-- 非流式 → [reflection_node]
                            |
                            |--- pass / 达上限 → END
                            |--- fail → 回 llm_node (循环重写)
                     |
                     v
                    END
```

多智能体图（步 7~8）是同一个套路，把「检索节点」换成「能自己回答的智能体」，两个助手并行查完由 `integration` 节点综合。

## 8 步学习卡片

每张卡片：**本步学什么 / 核心概念 / 源码+笔记 / 里程碑 / 建议时长**。

### 第 1 步 · 对话模型入门

- **学什么**：怎么用 LangChain 调一个大模型来对话（同步 + 流式）。
- **核心概念**：`ChatOpenAI`、`SystemMessage`/`HumanMessage`/`AIMessage`、`.invoke()`、`.stream()`、`.bind()`。
- **源码**：[../backend/services/llm_service.py](../backend/services/llm_service.py) ｜ **笔记**：[01-llm_service-对话模型入门](01-llm_service-对话模型入门.md)
- **里程碑**：能解释「字典消息 → 消息对象 → 调模型 → 取 content」这条链。
- **时长**：约 45 分钟。

### 第 2 步 · 向量化

- **学什么**：怎么把文本变成向量（给检索用）。
- **核心概念**：`OpenAIEmbeddings`、`embed_query`（查一条）、`embed_documents`（入库一批）。
- **源码**：[../backend/services/embedding_service.py](../backend/services/embedding_service.py) ｜ **笔记**：[02-embedding_service-向量化](02-embedding_service-向量化.md)
- **里程碑**：分清「查询向量化」和「文档向量化」两种用途。
- **时长**：约 30 分钟。

### 第 3 步 · 文本切分

- **学什么**：怎么把长文档切成带重叠的小块（切完才好向量化、检索）。
- **核心概念**：`RecursiveCharacterTextSplitter`、`chunk_size`/`chunk_overlap`、`separators`、`create_documents`、`page_content`。
- **源码**：[../backend/core/utils.py](../backend/core/utils.py)（`split_text` 函数） ｜ **笔记**：[03-utils-文本切分](03-utils-文本切分.md)
- **里程碑**：理解为什么要「重叠」、分隔符优先级怎么影响切分质量。
- **时长**：约 30 分钟。

> 到这里 LangChain 三个组件全过完了。接下来进入 LangGraph。

### 第 4 步 · LangGraph 状态与节点

- **学什么**：LangGraph 的核心抽象——状态（`TypedDict`）和节点（函数）怎么写。
- **核心概念**：`GraphState(TypedDict, total=False)`、节点函数 `def node(self, state) -> dict`、`get_stream_writer()`、路由函数。
- **源码**：[../backend/nodes.py](../backend/nodes.py) ｜ **笔记**：[04-nodes-LangGraph状态与节点](04-nodes-LangGraph状态与节点.md)
- **里程碑**：能说出「节点只返回增量 dict，LangGraph 自动合并进状态」。
- **时长**：约 1 小时。

### 第 5 步 · LangGraph 构图与编译

- **学什么**：怎么把节点用「边」连成一张图，再编译成可运行的对象。重点体会**并行扇出**和**反思循环**。
- **核心概念**：`StateGraph`、`add_node`、`add_edge`（含 `START`/`END`）、`add_conditional_edges`、`compile()`、路由返回列表=并行。
- **源码**：[../backend/graph.py](../backend/graph.py) ｜ **笔记**：[05-graph-LangGraph构图与编译](05-graph-LangGraph构图与编译.md)
- **里程碑**：看懂「为什么多条边指向同一节点会自动等待」。
- **时长**：约 45 分钟。

### 第 6 步 · 运行图（同步 + 流式）

- **学什么**：编译好的图怎么被调用——同步 `invoke` 和异步流式 `astream`，以及 SSE 怎么把字推给前端。
- **核心概念**：`invoke`、`astream(stream_mode=["updates","custom"])`、双流（节点进度 updates + 字符增量 custom）、SSE 帧。
- **源码**：[../backend/api.py](../backend/api.py)（`/chat` 和 `/chat/stream`） ｜ **笔记**：[06-api-运行图同步流式](06-api-运行图同步流式.md)
- **里程碑**：能解释「updates 流和 custom 流各管什么、怎么交替输出」。
- **时长**：约 1 小时。

> 单路图全通。最后进入多智能体，它和单路图用**同一套 API**，只是拓扑更花。

### 第 7 步 · 多智能体节点

- **学什么**：把「检索节点」升级成「会自己回答的智能体」，两个助手并行查再整合。
- **核心概念**：`MultiAgentState`、`rag_agent`/`web_agent`（对称设计）、`integration`（整合+流式）、Agent=工具+LLM。
- **源码**：[../backend/multiagent/nodes.py](../backend/multiagent/nodes.py) ｜ **笔记**：[07-multiagent-nodes-多智能体节点](07-multiagent-nodes-多智能体节点.md)
- **里程碑**：理解「为什么两个 agent 写得几乎一模一样」。
- **时长**：约 45 分钟。

### 第 8 步 · 多智能体构图

- **学什么**：把多智能体节点连成「并行扇出 → 汇合整合」的图。学习路径的终点。
- **核心概念**：fan-out（路由返回列表触发并行）、fan-in（多条汇合边触发等待）、与单路图同构、为什么省掉反思。
- **源码**：[../backend/multiagent/graph.py](../backend/multiagent/graph.py) ｜ **笔记**：[08-multiagent-graph-多智能体构图](08-multiagent-graph-多智能体构图.md)
- **里程碑**：能独立说出「要加第三个助手改哪三处」。
- **时长**：约 45 分钟。

# 配置与启动：应用怎么被装配起来（config.py + main.py）

> 读完 01~08 你已经懂了「单个零件」和「把零件连成图」。这一篇回答另一个问题：**这些零件，是在哪里被一次性创建出来、组装好、再启动服务的？** 答案在两个文件：[core/config.py](../backend/core/config.py)（配置）和 [main.py](../backend/main.py)（启动装配）。

## 它在整个项目里的位置

``+                          ┌─────────────────────────────────────┐
   .env（密钥/地址）  ───► │  core/config.py  Settings(单例)      │
                          └───────────────┬─────────────────────┘
                                          │ get_settings()
                          ┌───────────────▼─────────────────────┐
                          │  main.py  lifespan()                │  ◄── 启动时一次性
                          │   构造 LLM/Embedding/Qdrant/Neo4j/   │      创建所有服务
                          │   RAG/Graph/WebSearch，挂到 app.state │
                          └───────────────┬─────────────────────┘
                                          │ app.state
                          ┌───────────────▼─────────────────────┐
                          │  api.py  路由从 request.app.state 取  │  ◄── 每次请求复用
                          └─────────────────────────────────────┘
```

一句话：**配置集中在 config.py，服务实例集中在 main.py，请求时只复用、不重建。**

## 一、core/config.py：统一配置入口

### 它解决什么问题

项目有十几个可调参数（大模型地址、向量库地址、分块大小、端口……）。如果散落在各处用 `os.environ` 读，会乱、会重复、改一个地方不知道影响谁。本项目用一个 **强类型的配置类** 统一管理。

### 怎么读配置：pydantic-settings

[config.py](../backend/core/config.py) 的核心是一个继承 `BaseSettings` 的类：

```python
from pydantic_settings import BaseSettings, SettingsConfigDict

class Settings(BaseSettings):
    model_config = SettingsConfigDict(
        env_file=str(BASE_DIR / ".env"),   # 从 .env 文件读
        env_file_encoding="utf-8",
        case_sensitive=False,              # 环境变量大小写不敏感
        extra="ignore",                    # .env 里多余的键忽略
    )

    llm_base_url: str = "https://open.bigmodel.cn/api/paas/v4/"
    llm_model: str = "glm-5.2"
    embedding_dimension: int = 2048
    qdrant_top_k: int = 5
    neo4j_uri: str = "bolt://localhost:7687"
    ...
```

- `llm_model` 这个字段名，会自动去 `.env` 里找 `LLM_MODEL`（大小写不敏感）。
- `= "glm-5.2"` 是**默认值**：`.env` 没写就用它，写了就用写的。
- 类型 `int` / `float` / `str` 会让 pydantic 自动把字符串环境变量转成对应类型。

### 改配置的正确方式

**不要**在代码里硬编码，而是改 [backend/.env](../backend/.env.example)（参考 `.env.example`）。比如要换大模型：

```ini
LLM_MODEL=glm-4.6
LLM_BASE_URL=http://127.0.0.1:15721/v1
QDRANT_TOP_K=8
```

### 单例：lru_cache

```python
@lru_cache
def get_settings() -> Settings:
    return Settings()
```

- `@lru_cache` 让 `get_settings()` **只构造一次**，之后都返回同一个对象。
- 好处：`.env` 只读一次、配置全校验一次，全项目共享同一份配置。业务代码都用 `get_settings()` 取，而不是 `Settings()`。

### 把字符串拆成列表：@property

```python
cors_origins: str = "http://localhost:5173,http://localhost:4173"

@property
def cors_origins_list(self) -> list[str]:
    return [o.strip() for o in self.cors_origins.split(",") if o.strip()]
```

配置里存成逗号分隔的字符串（因为环境变量只能是字符串），用时通过 `settings.cors_origins_list` 拿到列表。这是 `@property`（见 [00](00-Python语法速成.md#13-property--把方法当属性用)）的典型用法。

### 配置项速查

| 分组 | 关键字段 | 作用 |
|------|----------|------|
| LLM | `llm_model` `llm_base_url` `llm_temperature` | 对话模型与参数 |
| Embedding | `embedding_model` `embedding_dimension` | 向量化模型与维度 |
| Qdrant | `qdrant_url` `qdrant_collection` `qdrant_top_k` `qdrant_score_threshold` | 向量库地址/集合名/检索条数/相关度阈值 |
| Neo4j | `neo4j_uri` `neo4j_user` `neo4j_password` | 图谱库连接 |
| 联网 | `tavily_api_key` `tavily_max_results` | Tavily 搜索（留空则不可用） |
| RAG | `chunk_size` `chunk_overlap` `max_reflection_iterations` | 分块大小/重叠/最大反思次数 |
| 应用 | `app_host` `app_port` `cors_origins` `log_level` | 监听地址/端口/跨域/日志级别 |

## 二、main.py：启动装配（lifespan）

### FastAPI 应用的三件套

```python
app = FastAPI(title="...", version="1.0.0", lifespan=lifespan)

app.add_middleware(CORSMiddleware, ...)        # ① 中间件：跨域
app.include_router(api_router, prefix="/api")  # ② 路由：把 api.py 的接口挂到 /api 下

if __name__ == "__main__":
    uvicorn.run("main:app", host=..., port=...) # ③ 启动服务器
```

### lifespan：启动建、停止拆

`lifespan` 是一个**异步上下文管理器**（`@asynccontextmanager`），FastAPI 在应用启动时跑 `yield` 之前的代码，在停止时跑 `yield` 之后的代码：

```python
@asynccontextmanager
async def lifespan(app: FastAPI):
    # —— 启动时：创建所有服务单例 ——
    llm = LLMService(settings)
    embedding = EmbeddingService(settings)
    qdrant = QdrantStore(settings, embedding)
    neo4j = Neo4jStore(settings)
    rag = RagService(qdrant=qdrant, neo4j=neo4j, llm=llm, settings=settings)

    qdrant.ensure_collection()   # 尽力预热，失败不阻塞
    neo4j.verify()

    app.state.graph = build_graph(llm, rag, settings)
    app.state.multi_agent_graph = build_multi_agent_graph(llm, rag, web, settings)
    ...
    yield                          # ◄── 应用运行期间停在这里
    # —— 停止时：关闭连接 ——
    neo4j.close()
```

几个关键点：

1. **依赖注入顺序**：`embedding` 要先于 `qdrant` 创建（向量库依赖它做向量化）；`rag` 要等 `qdrant`/`neo4j`/`llm` 都建好。
2. **挂到 app.state**：所有服务实例挂在 `app.state` 上，每个请求都能通过 `request.app.state.xxx` 拿到，**全程复用同一个对象**，不会每次请求都重新建连接。
3. **优雅降级**：`try ... except` 包住 `ensure_collection()` 和 `verify()`。存储没起来只是打个 warning，**不阻止启动**——这样前端能在 `/health` 看到降级状态。
4. **预热**：`ensure_collection()` 会在 Qdrant 不存在时按向量维度建集合；`verify()` 检查 Neo4j 连通。

### CORS：跨域中间件

浏览器默认不让前端（跑在 `localhost:5173`）调不同源的后端（`localhost:8000`）。CORS 中间件就是显式放行：

```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=settings.cors_origins_list,  # 允许哪些前端地址
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

> 如果前端联调时浏览器报「CORS 错误」，先检查这里 `allow_origins` 有没有包含前端的地址。

### 启动命令

```powershell
# 方式一：直接跑（用 main.py 末尾的 uvicorn.run）
python main.py
# 方式二：开发模式热重载
uvicorn main:app --reload
```

- `main:app` 指「`main.py` 文件里的 `app` 变量」。
- `--reload` 改代码自动重启，开发时用。

这是读 main.py lifespan 最该记住的一张图——**谁依赖谁、创建顺序**：

## 修改建议

- **加一个新配置项**：在 `Settings` 类加字段+默认值，再到 `.env.example` 写说明，业务代码用 `get_settings().新字段` 读。三步缺一不可。
- **加一个新服务**：在 `lifespan` 启动段创建它、`app.state.xxx = ...` 挂上、停止段做清理（关闭连接）。然后路由就能用。
- **不要**在路由函数里 `Settings()` 或 `QdrantStore(...)` 新建对象——那会每次请求都建连接，又慢又泄漏资源。一律从 `request.app.state` 取。
